## 1. 系统概述

### 1.1 项目定位

西北师范大学社区团购系统。商家发布拼团活动，用户开团/参团→支付→成团→自提或快递→售后。

| 组件 | 技术 |

| 用户端 | uni-app H5 (yudao-mall-uniapp) |
| 数据库 | MySQL 8 |
| 缓存 | Redis |
| 容器 | Podman (MySQL/Redis/frontend) |

## 2. 架构模型

### 2.1 租户策略演变

| 阶段 | 模型 | 说明 |
|------|------|------|
| 初始 | 多租户完全隔离 | 每个商家一个 tenant（164, 165...），H5 写死 tenant-id=1 |
| **最终** | **单租户 + 部门数据权限** | 所有商家在 tenant=1 内，用 `dept_id` 隔离 |

### 2.2 单租户模型

```
tenant_id=1 (西北师大团购)
├── dept_id=200  商家001 (mall001 老板)
│   ├── dept_id=200  商家001 自提门店
│   ├── kefu001 (客服小张)
│   └── kefu0012 (客服小王)
└── dept_id=201  商家002 (mall002 老板)
    ├── dept_id=201  商家002 自提门店
    └── kefu002 (客服小李)
```

- **商家 = 部门**：`system_dept` 表的一个顶级节点就是一个商家
- **数据权限**：使用 yudao 内置 `DeptDataPermissionRule`，商家角色 `dataScope=DEPT_ONLY`
- **H5 用户**：MEMBER 类型，不经过 DeptDataPermissionRule 过滤，可以看到所有商家的活动
- **平台管理员 (admin)**：`dataScope=ALL`，不操作商城业务，只管系统配置

### 2.3 dept_id 字段

| 表 | 作用 | 自动填充 |
|---|------|----------|
| `product_spu` | 商品所属商家 | `ProductSpuServiceImpl.createSpu()` |
| `group_buy_activity` | 活动所属商家 | `GroupBuyActivityServiceImpl.createActivity()` |
| `trade_order` | 订单所属商家 | `TradeGroupBuyOrderHandler.beforeOrderCreate()`（从活动继承） |
| `trade_delivery_pick_up_store` | 自提门店所属商家 | `DeliveryPickUpStoreServiceImpl.createDeliveryPickUpStore()` |
| `promotion_kefu_conversation` | 客服会话所属商家 | `getOrCreateConversation()`（默认 dept_id=200） |

## 8. 客服模块

### 8.1 数据模型

```
promotion_kefu_conversation (会话)
├── user_id          → 用户
├── dept_id          → 商家部门 (200/201)
├── assigned_to      → 分配的客服 (轮询)
└── admin_unread..   → 未读数

promotion_kefu_message (消息)
├── conversation_id  → 会话
├── sender_id/type   → 发送者 (会员/管理员)
└── content_type     → 文本/商品/订单
```

### 8.2 轮询分配

新会话自动分配给**当前有效会话数最少**的客服。逻辑：
```sql
SELECT u.id FROM system_users u
JOIN system_user_role ur ON u.id = ur.user_id
JOIN system_role r ON ur.role_id = r.id
WHERE r.code = 'kefu_staff' AND u.dept_id = ?
ORDER BY (SELECT COUNT(*) FROM promotion_kefu_conversation
          WHERE assigned_to = u.id AND admin_deleted=0) ASC
LIMIT 1
```

若管理员回复未分配会话，自动归属给该管理员。

### 8.3 消息类型

| 类型 | 说明 |
|------|------|
| TEXT (1) | 文本消息 |
| PRODUCT (10) | 商品卡片消息 |
| ORDER (11) | 订单卡片消息 |

管理后台客服页面为三栏布局：会话列表 | 消息聊天 | 用户信息。

## 9. 启动方式

详见项目根目录 `start.md`：

```bash
cd /mnt/data_d/Projects/ruoyi
./restart-services.sh     # 一键启动 MySQL+Redis+后端+两个前端
```

访问：
- 管理后台：http://127.0.0.1:5174
- 用户端 H5：http://127.0.0.1:5175

初始化数据库（首次或重置）：执行 `sql/mysql/group-buy-single-tenant-init.sql`。

### 9.1 后端编译

```bash
# settings.xml 含阿里云镜像
podman run --rm --network host \
  -v /mnt/data_d/Projects/ruoyi/backend/ruoyi-vue-pro:/workspace \
  -v "$HOME/.m2/repository:/root/.m2/repository" \
  -v /tmp/opencode/settings.xml:/root/.m2/settings.xml \
  -w /workspace maven:3.9-eclipse-temurin-17 \
  mvn install -pl yudao-server -am -DskipTests -T 1C
```

### 9.2 重启后端（不重新编译）

```bash
fuser -k 48080/tcp; sleep 2
nohup java -jar /mnt/data_d/Projects/ruoyi/backend/ruoyi-vue-pro/yudao-server/target/yudao-server.jar \
  --spring.profiles.active=local --yudao.security.mock-enable=false > /tmp/yudao.log 2>&1 &
```
