## 1. 改造概述

### 1.1 原系统架构

| 组件 | 原系统 |
|------|--------|
| 后端框架 | ruoyi-vue-pro (Spring Boot 3 + MyBatis-Plus) |
| 管理后台前端 | yudao-ui-admin-vue3 (Vue 3 + Element Plus) |
| 用户端前端 | yudao-mall-uniapp (uni-app，支持 H5 / 小程序 / App) |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis 7 |

### 1.2 改造目标

在原有商城系统基础上，实现**社区团购系统**，核心流程：

```
商家发布拼团活动 → 用户发起/参与拼团 → 立即付款 → 拼团成功/失败
    ↓                              ↓
商家核销/自提 ← 发货/待收货 ← 生成 trade_order
```

支持能力：
- 多角色：超级管理员、商家（团长）、普通用户
- 多租户：每个商家独立数据隔离
- 团长制：商家即团长，负责开团与核销
- 自提：用户到指定自提点提货
- 立即付款：与 yudao 支付模块一致

### 1.3 改造范围

| 模块 | 改造内容 |
|------|----------|
| 数据库 | 新增 3 张拼团核心表；修复 18 张表 tenant_id；补 8 张缺失表；修复 DIY 模板字段 |
| 后端 | 新增独立模块 `yudao-module-groupbuy`（activity/head/member）；状态机；过期任务；核销 |
| 前端（管理端） | 菜单精简、首页实时数据、拼团活动/团单/核销页面 |
| 前端（用户端） | 启用 yudao-mall-uniapp，新增拼团列表/活动/我的拼团页面 |

### 1.4 项目 Git 结构

| 路径 | 类型 | 来源 |
|------|------|------|
| `backend/ruoyi-vue-pro` | submodule | github.com/YunaiV/ruoyi-vue-pro |

### 2.2 已完成的功能

- 管理端：发布/编辑/关闭拼团活动，查看团单，团长核销
- 后端：开团、参团、成团状态机、过期自动失败、库存预扣/回滚，已接入 trade order 与钱包支付回调
- 用户端：拼团活动列表/详情/我的团单接入，开团/参团进入标准订单确认与支付链路，H5 能正常构建

### 2.3 当前状态

完整闭环 **管理端发布 → 用户开团/参团 → 支付 → 发货 → 收货 → 拼团结束** 已在本地跑通：

1. 管理端发布/维护拼团活动。
2. 用户端活动详情点击开团/参团，进入 `trade/order/settlement` 与 `trade/order/create`。
3. `trade_order.type = 5`，订单写入 `group_buy_activity_id` / `group_buy_head_id`。
4. 钱包支付成功后，trade 支付回调触发 `GroupBuyHeadServiceImpl.onMemberPaid(orderId)`。
5. 满员后团头 `status = SUCCESS`，订单允许后台发货，用户可确认收货。

仍需继续完善：拼团失败后的自动退款、首页 DIY 装修内容、生产级短信/支付渠道配置。

## 3. 数据库改造

### 3.1 新增拼团核心表

| 表名 | 说明 |
|------|------|
| `group_buy_activity` | 拼团活动（商品、价格、成团人数、库存、时间） |
| `group_buy_head` | 团单（谁开的团、当前人数、状态、过期时间） |
| `group_buy_member` | 团成员（用户、是否团长、订单号、状态） |

同时 `trade_order` 扩展了 `group_buy_head_id`、`group_buy_activity_id` 字段，用于把普通订单与拼团关联。

### 3.2 其他改造

- 18 张 pay/trade/promotion/member 表添加 `tenant_id`
- 8 张缺失表补建（如 `trade_delivery_express_template` 等）
- `product_sku.pic_url`、`promotion_coupon_template` 等字段修复
- `promotion_diy_template.property` / `promotion_diy_page.property` 从 `varchar(255)` 改为 `TEXT`
- MySQL 字符集从 `latin1` 改为 `utf8mb4`

### 3.3 迁移脚本

| 文件 | 作用 |
|------|------|
| `sql/ruoyi-vue-pro.sql` | yudao 原始库结构 |
| `sql/mysql/group-buy-migration.sql` | 菜单清理、多租户字段、缺失表 |
| `sql/mysql/group-buy-ddl.sql` | 拼团 3 张新表 + trade_order 扩展 + 菜单权限 |
| `sql/mysql/group-buy-test-data-v2.sql` | 拼团测试数据（6 活动 + 5 团单 + 12 成员） |
| `sql/mysql/group-buy-uniapp-fix.sql` | **修复 uniapp H5 启动**：DIY 字段 + 默认模板/页面/tabBar |
| `sql/mysql/group-buy-menu-cleanup.sql` | 删除无用菜单 |

## 4. 后端改造

### 4.1 新增模块

```
backend/ruoyi-vue-pro/yudao-module-mall/yudao-module-groupbuy/
├── controller/admin/activity/   # 拼团活动管理 API
├── controller/admin/head/       # 团单管理 API
├── controller/admin/verify/     # 团长核销 API
├── controller/app/activity/     # 用户端活动 API
├── controller/app/head/         # 用户端开团/参团 API
├── dal/dataobject/              # GroupBuyActivityDO / GroupBuyHeadDO / GroupBuyMemberDO
├── dal/mysql/                   # Mapper
├── service/                     # 业务逻辑 + 过期任务
└── enums/                       # 状态枚举
```

### 4.2 状态机

```
拼团中(IN_PROGRESS) ──满员──▶ 已成团(SUCCESS)
         │
         └──过期/失败──▶ 已失败(FAILED) ──▶ 回滚库存
```

- 满员：最后一个支付成功的成员触发 `markSuccess`
- 过期：`GroupBuyHeadExpireJob` 定时扫描，超期未成团则标记失败并回滚库存
- 失败成员的退款：目前仅打印日志，未完整接入 pay 模块

### 4.3 关键 API

| 端点 | 说明 |
|------|------|
| `GET /app-api/trade/order/settlement` | 拼团订单确认，传 `groupBuyActivityId` / `groupBuyHeadId` |
| `POST /app-api/trade/order/create` | 创建拼团订单；无 `groupBuyHeadId` 为开团，有则为参团 |
| `POST /app-api/pay/order/submit` | 钱包支付拼团订单 |
| `GET /app-api/group-buy/activity/list` | 活动列表 |
| `GET /admin-api/group-buy/activity/page` | 管理端活动分页 |
| `GET /admin-api/group-buy/head/page` | 管理端团单分页 |
| `POST /admin-api/group-buy/verify/verify` | 团长核销 |

### 4.4 已知未接入点

- 拼团失败后的退款链路仍未完整接入 pay refund。
- `/app-api/group-buy/head/open`、`/join` 保留为旧接口，不作为主流程入口；主流程统一走 trade order。

## 6. 启动方式

详见项目根目录 `start.md`。核心命令：

```bash
# 1. 启动 MySQL + Redis
podman start ruoyi-mysql ruoyi-redis

# 2. 启动后端 + 两个前端（已放后台）
cd /mnt/data_d/Projects/ruoyi
./restart-services.sh

# 3. 访问
# 管理后台：http://localhost:5174
# 用户端 H5：http://localhost:5175
```

如果数据库还没初始化，按 `start.md` 第 2 步导入全部 SQL，其中**一定要执行** `group-buy-uniapp-fix.sql`，否则 uniapp H5 会因缺少 DIY 模板而白屏。

## 7. 遇到的问题与解决方案

### 7.1 构建与环境

| # | 问题 | 解决方案 |
|---|------|----------|
| 1 | FUSE overlay 导致 `mvn clean` 失败 | 改用 `mvn install` 跳过 clean |
| 2 | MySQL 字符集 latin1 导致菜单乱码 | 添加 `custom.cnf` 强制 utf8mb4 |
| 3 | uniapp CLI 找不到 `src/manifest.json` | 启动时显式传入 `UNI_INPUT_DIR=/workspace` |
| 4 | uniapp H5 白屏，报 `$api before initialization` | 修复 `request/store/api` 循环依赖；延迟加载 platform/store |
| 5 | uniapp H5 报 `uni is not defined` | `platform/index.js` 顶层 `uni.*` 改为安全访问 |
| 6 | uniapp 首页提示“未找到模板” | 扩 `property` 字段为 TEXT；补默认 DIY 模板与页面 |

### 7.2 多租户

| # | 问题 | 解决方案 |
|---|------|----------|
| 7 | 18 张表缺少 `tenant_id` | 批量 ALTER TABLE |
| 8 | 缺失 8 张表 | 创建建表语句 |
| 9 | GroupBuy DO 继承 BaseDO 无法隔离 | 改为继承 TenantBaseDO |
| 10 | 商家角色跨租户访问 | 添加 TenantSecurityWebFilter |

### 7.3 代码 Bug

| # | 问题 | 解决方案 |
|---|------|----------|
| 11 | 统计页 NPE（无 times 参数） | 3 个 Controller 添加 null 检查 + 默认值 |
| 12 | 首页数字写死 | 改为 4 个实时 API |
| 13 | 快捷入口 404 | 修正为叶子路由路径 |
